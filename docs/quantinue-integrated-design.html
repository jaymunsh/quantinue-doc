Quantinue · 2차 (MVP-2) · v4.0

AI가 제안하고, AI가 반박하고,
코드가 막는다.

공시·뉴스·기술지표·매크로를 읽어 AI가 매수를 제안하면, 또 다른 AI가 적대적으로 반박하고, 결정론적 코드가 리스크를 계산해 페이퍼 계좌로 집행한 뒤 5영업일 후 복기한다. 2차는 이 사이클을 사람 손 없이 자동으로 돌리고, 관리자·유저 화면을 얹어 AI 페이퍼 펀드 서비스로 완성한다. LLM이 검증 없이 돈에 닿는 경로는 0개.

파이프라인11단계 투자 성향공격형·안전형 계좌페이퍼·다계좌 운영무인 자동 LLMOpenAI

Decisions

2차 확정 사항

본격 개발 전 대화로 확정한 결정 8건(R1~R8). 이 결정들이 아래 모든 섹션의 전제다. 세부 근거·대안 검토는 docs/mvp2-planning/phase1-decisions.md에 보존.

R1

2차 = AI 페이퍼 펀드 서비스

사람 손 없이 도는 무인 운영 위에 두 개의 화면을 얹는다 — 관리자(펀드매니저 ERP: 계좌 CRUD·운영·비용·리포트)와 유저(조회 전용 포털). 리서치 신호(01~08)는 사이클당 1회 공용 생성, 계좌는 성향별 파라미터로 구독·실행(09~10)하므로 LLM 비용이 계좌 수와 무관하게 고정된다. 실계좌 전환은 범위 외(페이퍼 전용).

성향 공격형·안전형유저 read-only계좌 생성 관리자만
R2

앱 내장 스케줄러 · cycle_ts 슬롯 멱등

FastAPI 앱 안의 백그라운드 루프가 역할별 실행 창(뉴욕 시간)에 파이프라인을 자동 구동한다. cycle_ts는 실행 시각이 아니라 주기 슬롯 경계로 내림한 값이라, 같은 슬롯을 몇 번 실행해도 UNIQUE(ticker, cycle_ts)가 중복을 막는다(멱등). 슬롯은 계산값이라 마스터 테이블을 두지 않고, 실행 기록은 기존 pipeline_runs가 담당.

배포 로컬 → AWS주문 계열 개장 중만
R3

판단 방어선 3종 전부 구현

1차에서 하드코딩 껍데기로 남았던 게이트를 실제로 작동시킨다 — 출처등급(소셜·미등록 매체는 LLM 도달 전 사전 차단)·07 게이트 3종(신뢰도 낮은 뉴스 투표 박탈, 위험 국면 감점, 강한 악재 차단)·snapshot 실전달(08이 07 박제값을 실제로 검증). 모든 문턱은 config 소유 ⚙️ 운영 실측 후 조정 가능.

source_trust ≥0.55강악재 ≤0.15매크로 감점 −0.40snapshot 오차 2%Critic 0.70과신 0.90→0.80
R4

LLM 3단 실행 모드

mock(가짜 점수 · 단위 테스트) → 로컬 qwen(이 PC의 MLX · 개발·통합 테스트, 무료) → OpenAI gpt-4o-mini(운영). 모델 전환은 config 몇 줄. 단 점수 눈금은 모델마다 다르므로 로컬은 배관 검증용이고, 문턱값 캘리브레이션은 운영 모델 실측으로 한다.

R5

신호 품질 7건 (실무 개정 3건 포함)

reason를 점수별 근거 객체(JSONB)로 확장(#23) — 유저 화면 "거래 이유"의 원천. 대표 기사는 관련성 필터(오귀속 차단) 통과분 중 importance×신뢰무게 최상위로 개정. Form 4는 템플릿 조립하되 자발 매수(코드 P)만 유의미 신호로. consensus는 설명용 지표로만(게이트 아님 — 좁은 깔때기 방지). peak_importance 계산·halted 체크도 구현.

R6

학습 루프 · 비용 가드

T+5(영업일) 회고 채점을 자동화하고 lesson을 생성해, 07 전략이 같은 종목 최근 lesson을 프롬프트에 주입받는다 — "회고가 다음 판단을 개선한다"는 서사의 완성. LLM 호출을 전부 기록해 관리자 대시보드에 사용량·비용을 노출하고, 일일 예산 상한 초과 시 호출을 멈춘다(그 사이클 신규 매수 보류).

R7

실무 보강 6건

1차에 빠져 있던 실무 기본기 — ① 매도·보유 재평가(보유 종목 상시 분석 + 청산 3층: 가격·시간·논지 붕괴) ② 포트폴리오 서킷브레이커(일일 손실 한도 도달 시 당일 신규 매수 중단) ③ 텔레그램 알림 ④ NYSE 휴장일 캘린더 ⑤ 계좌 정합 대사 ⑥ SPY 벤치마크. ①이 07 파이프라인 구조에 손대는 최대 설계 작업.

R8

스크리닝 깔때기 원설계 복원

1차 코드의 임의 축소(50→20→20)를 되돌려, 주1회 시총 상위 2,000 감시 → 매일 유동성 필터 후 오늘의 50 → LLM 심층은 상위 20 ∪ 보유 종목. 촉매 기반 전략이 재료를 놓치지 않게 감시망을 넓힌다.

Conventions

공통 컨벤션 — 어휘·타입 규칙

이 8규칙이 문서 전체 어휘의 정본이다. 스키마·전달 JSON·코드 표기가 어긋나면 이 규칙이 이긴다. (1차에서 확정·계승, 규칙 4는 2차에서 개정.)

  1. Boolean은 BOOLEAN 타입 + is_/has_ 접두. INT 0/1 금지. 상태는 is_, 보유·존재는 has_ (is_hard_blocked·has_signal).
  2. ENUM은 영어 소문자 snake_case, 물리 타입은 TEXT. Postgres 네이티브 ENUM·참조 테이블 FK를 쓰지 않고, 허용값 검증 1차 책임은 백엔드(Pydantic Literal). event_type 허용값 정본 = ontology.py.
  3. 시각은 *_at = TIMESTAMPTZ(UTC 저장, 표시만 KST), 날짜는 *_date = DATE.
  4. 점수는 전부 0~1. DB는 NUMERIC + CHECK(0~1), 표기·전달은 소수 3자리. 2차 개정 — 1차의 "NUMERIC(4,3) 예외 없음"에서 물리 정밀도 강제는 해제(값 범위만 강제, 코드 현실 수용).
  5. 전달 JSON 필드명 = DB 컬럼명. 단계마다 이름을 바꾸지 않는다(개명 금지) — 추적·디버깅의 기준.
  6. 이름 규칙 — 원본 원장은 자연 명사(tb_news), 집계 전달은 _signal 접미(tb_news_signal), 뷰는 v_ 접두.
  7. 신호(_signal) 테이블은 append 불변. update 금지 — 최신 상태는 (ticker, cycle_ts DESC) 뷰/인덱스로 읽는다.
  8. PK 규칙 — 시각 컬럼·ticker 단독을 PK로 쓰지 않는다. 사이클 append 테이블은 대리키 PK(tns_id·tds_id) + UNIQUE(ticker, cycle_ts)(멱등), 일봉 테이블은 (날짜, ticker) 자연 복합 PK.
