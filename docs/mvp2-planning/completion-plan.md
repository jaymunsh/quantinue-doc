## 2. 순서를 이렇게 잡은 이유 (셋만)

1. **②가 ③보다 먼저다.** 계좌별 읽기 계층을 한 번만 만든다. 관리자 총람과
   유저 홈이 같은 리더를 쓰면 두 화면이 다른 숫자를 말할 수 없다.
2. **④는 ②③ 뒤다.** 화면이 유령 계좌를 보는 동안 무인 관측을 켜면, 무엇이
   틀어졌는지 화면으로 읽을 수 없다.
3. **⑨⑩은 뒤다.** 둘 다 "지금 안 터지는 것"이고, ⑨는 전환 결정이 트리거,
   ⑩은 관측 종료가 트리거다. 트리거 없이 당기면 그게 유령이다.
