# AWS 이전 검토 — 얻는 것 / 치르는 것 / 선행 조건 / 권고 (2026-07-21)

> 지금은 맥 한 대에서 사람이 켜고 끄는 운용이다. 이 문서는 "AWS로 옮기면
> 무엇이 달라지고, 옮긴다면 언제·어떤 순서인가"에 답한다.
> 형식은 `future-roadmap.md`를 따른다. LLM 사용처 전수는
> `llm-usage-guide.md`가 정본이다.
>
> **운영 전제(2026-07-21 확인)**: OpenAI 크레딧 잔액 ≈ **$90**, 목표 운영
> 지평 ≈ **10일**. 이 두 숫자가 아래 판단의 축이다.

---

## 0. 결론 먼저 (2026-07-21 재판정)

~~운영 지평이 ~10일이라면 AWS 이전은 권하지 않는다~~ — **전제가 바뀌어
뒤집는다.** 원 결론의 대체재는 "맥을 10일간 안 끄고 두는 것"이었는데,
실제 운용자는 **이동이 잦다**(2026-07-21 확인). 노트북이 이동하면 전원·
네트워크가 끊기고, 그 공백은 알림 체계상 **감지도 안 된다**(보내는 주체가
앱이라 앱이 죽으면 침묵 — 침묵이 신호이긴 하나 신호가 "하루 뒤"에 온다).
상시 가동이 안 되는 맥은 무인 완주의 관측 도구로 부적합하다.

**따라서 AWS 이전을 진행한다. 단 순서는 그대로다** — LLM 쪽 선행 작업을
건너뛰고 옮기면 예산 없이 유료 API가 무인으로 도는 상태가 되기 때문이다:

1. **LLM 예산 배선(⑨)** — 로컬에서, 이전과 무관하게 먼저
2. **openai 리허설** — 로컬 8021에서 모드 한 줄로 (§3-3 재확인 포함)
3. **AWS 이전** — 그 다음. 이전 직후 며칠은 관측 초기화로 간주

크레딧 $90는 10일 LLM 비용($1~5 수준, §2)의 수십 배라 비용은 제약이 아니다.
인프라 비용(월 $20~40 자릿수)이 LLM보다 크다는 점만 인지할 것.

## 1. 의존 사슬 — 이것부터 확인했다

```
AWS 이전
 └─ 로컬 LLM(oMLX/Qwen3.6-35B) 사용 불가 — 맥 전용 (Apple Silicon MLX)
     └─ llm_mode=openai 강제
         └─ 호출당 비용 발생
             └─ LLM 예산 배선(open-items §3-1)이 선택 → 선행 필수로 승격
                 └─ §3-3 재확인 2건(retries·성향 격차)이 딸려 온다
```

즉 "AWS 이전"은 인프라 작업이 아니라 **LLM 비용 통제 작업이 앞에 붙은
작업**이다. 거꾸로 말하면 — **openai 전환을 AWS 없이 먼저 하면**, 이 사슬의
아래쪽(예산 배선·재확인)을 로컬에서 안전하게 소화할 수 있다. AWS는 그 뒤
순수 인프라 작업으로 홀가분해진다.

## 2. 비용 실산 — $90는 얼마나 버티나

`llm-usage-guide.md` 기준 하루 콜: STRATEGY 40~60 + CRITIC 수~수십 =
**최대 ~100콜/일**, 하루 입력 ~15만 토큰 + 출력 ~3만 토큰 자릿수.

| 모델 (성격) | 대략 요율 (in/out, 1M 토큰) | 하루 | 10일 |
|---|---|---|---|
| gpt-4o-mini (**경량** — 추론모델 아님) | $0.15 / $0.60 | ~$0.04 | **~$0.4** |
| o4-mini급 (추론 tier) | ~$0.55 / ~$2.20 (+추론 토큰이 출력으로 과금 ×3~5) | ~$0.2~0.5 | **~$2~5** |
| 프런티어급 | ~$2.5 / ~$10 | ~$1~3 | ~$10~30 |

> ⚠️ 요율은 2026-07 서드파티 집계 기준 자릿수다. **전환 직전에 공식 가격표로
> 재확인할 것.** 단 결론은 요율이 배로 틀려도 안 바뀐다 — $90 대비 한 자릿수다.

**읽는 법 둘:**

1. **"gpt-4o-mini를 모든 콜에 쓰면 과소비"는 성립하지 않는다** — 방향이
   반대다. gpt-4o-mini는 추론모델이 아니라 **최저가 경량 모델**이고, 하루
   4센트다. 걱정할 것은 과소비가 아니라 **과소성능**이다: 로컬 35B가 하던
   증거 종합·서사 생성을 최저가 모델이 같은 질로 하는지가 리스크다.
2. **$90 / 10일 = 하루 $9 여유** — 추론 tier를 전 콜에 써도 예산의 5%다.
   비용은 모델 선택의 제약이 아니다. `budget.daily_llm_usd`(현재 3.0)는
   그래도 배선한다 — 예산이 남아도 **폭주(버그로 인한 무한 재시도·중복
   호출)를 끊는 장치**는 있어야 유료 API를 무인으로 돌릴 수 있다.

## 3. 얻는 것 / 치르는 것 / 새로 필요한 것

| 얻는 것 | 비고 |
|---|---|
| 24시간 가동 — 무인 완주 조건이 구조적으로 충족 | 맥 꺼짐 사각지대 소멸 |
| 알림 신뢰도 상승 | "침묵 = 맥이 꺼졌나?"의 해석 부담 소멸 |
| 운영과 개발 머신 분리 | 지금은 코드 작업 실수가 관측을 오염시킬 수 있다(8020/8021 수동 분리) |

| 치르는 것 | 비고 |
|---|---|
| 로컬 LLM의 무료 추론 | oMLX/Qwen 35B — 맥 전용. openai 전환 강제(§1) |
| 지금의 단순함 | docker 1 + 앱 1 + .env → EC2/컨테이너 + 비밀관리 + 배포 파이프라인 |
| 인프라 비용 | EC2 소형 + DB로 월 $20~40 자릿수. LLM보다 이게 크다 |
| 이전 기간의 관측 공백·초기 결함 | 새 환경 첫 며칠은 무인 기록으로 못 친다 |

| 새로 필요한 것 | 판단 |
|---|---|
| **LLM 예산 배선** (⑨, `tb_llm_usage` + `daily_llm_usd` 강제) | **선행 필수** — AWS 없이도 openai 전환 순간 필요 |
| 비밀 관리 (.env → SSM Parameter Store/Secrets Manager) | 이전 시 필수. 지금 구조(.env)는 단일 맥 전제 |
| DB — RDS vs EC2 내 컨테이너 | 규모(131MB, 하루 수 MB)로는 **컨테이너로 충분**. RDS는 운영 지속 결정 후 |
| 배포 방식 | 최소로: docker compose + git pull. CI/CD는 유령 |
| Alpaca 키가 다른 IP에서 도는 것 | Basic 플랜 IP 제약 여부 확인 필요. 데이터 전용 키라 리스크 낮음 |
| 시간대 처리 재점검 | 앱은 뉴욕 날짜 기준 슬롯 — 서버 TZ가 바뀌어도 코드가 UTC/NY 기준인지 확인 |

## 4. 권고 순서

**1단계 (지금, AWS와 무관): openai 전환 준비 — 로컬에서**
1. LLM 예산 배선(⑨): 매 콜 `tb_llm_usage` 기록 + `daily_llm_usd` 초과 시
   호출 스킵("안 사는" 쪽). TDD, 스키마는 이미 있다.
2. openai 경로에 model_settings 부여(max_tokens·temperature — 지금은
   로컬 경로에만 있다, `llm-usage-guide.md` §2).
3. 모델 선정: STRATEGY·CRITIC 공용으로 **추론 tier(o4-mini급)부터 시작**
   권장 — 비용이 제약이 아니므로(§2) 판단 질을 먼저 확보하고, 낮추는 실험은
   `tb_llm_usage` 데이터가 쌓인 뒤 근거를 갖고 한다.
4. `.env`에서 `QUANTINUE_LLM_MODE=openai`로 리허설 한 바퀴(8021, 일회용
   슬롯) — §3-3 재확인 2건(retries 동작·성향 격차)을 여기서 본다.

**2단계 (1단계 완료 직후): AWS 이전** *(원안의 "10일 로컬 관측"은 이동 잦음
확인으로 폐기 — §0 재판정)*
- 순수 인프라 작업이다: EC2 + docker compose(앱+DB) + SSM 비밀 +
  스냅샷 백업. LLM 쪽은 1단계에서 이미 끝나 있다.
- 이전 직후 며칠은 관측 초기화로 간주하고 실패 패턴을 다시 본다.
- 이전 전까지의 로컬 운용은 최선 노력으로만 — 전원 유지 + 텔레그램 침묵을
  신호로 읽는다. 끊김 즉시 감지가 필요하면 외부 데드맨 스위치(예:
  healthchecks.io — 앱이 주기 ping, 끊기면 그쪽이 알림)를 임시로 붙일 수
  있으나, AWS 이전이 근본 해결이므로 이전 일정이 짧으면 안 만든다(유령 방지).

## 5. 이 문서가 답하지 않은 것

- **모델 최종 선정** — 리허설(1단계 4)과 `tb_llm_usage` 실측이 답한다.
  후보 비교는 `llm-usage-guide.md` §3.
- **AWS 세부 아키텍처** — 3단계 착수 시점에 그때의 규모로 다시 그린다.
  지금 그리면 유령이다.
- **AWS 세부 견적** — EC2 인스턴스 크기·리전·DB 형태는 2단계 착수 시점에
  실측 사양(앱 24MB·DB 201MB·디스크 131MB)으로 정한다. 지금 확실한 것은
  "가장 작은 축에 든다"는 것뿐이다.
