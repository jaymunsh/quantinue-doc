## 3. 전환 전에 반드시 할 것 (조건부 필수)

### 3-1. LLM 예산 배선 — `tb_llm_usage` + `budget.daily_llm_usd`

**지금 안 해도 되는 이유**: 로컬 LLM(oMLX·Qwen3.6-35B)이라 호출 비용이 0이다.

**해야 하는 시점**: ① openai 모드 전환 ② 인트라데이 다회 사이클(R3).
둘 중 하나라도 하면 콜 수가 종목×성향×사이클로 곱해진다.

**내용**: 모든 LLM 호출을 `tb_llm_usage`(호출처·모델·토큰·추정비용)에 기록 →
`budget.daily_llm_usd`(현재 3.0, 선언만 있음) 초과 시 호출 스킵.
**초과 시 동작은 "판단 없이 사는" 쪽이 아니라 "안 사는" 쪽이어야 한다.**

### 3-2. 거래정지 종목 제출 차단 가드 (R1 전제)

구 `role_10`의 halted 가드가 서비스와 함께 죽었다. 로컬 시뮬(D1·D2)에선
제출 대상이 없어 무해하지만, **페이퍼 전환(R1) 때 다시 필요하다.**
덮은 척하지 않고 적어둔다.

### 3-3. openai 전환 시 재확인 2건

- **`retries`**: LOCAL 경로가 `retries=0`으로 굳어 있던 것을 config 소유로
  복원하고 종목 단위 격리를 넣었다(`provider.py`). openai에서 재시도 동작이
  의도대로인지 확인.
- **성향 격차**: 모델이 같은 값을 낸 뒤 보수형에 일괄 `-0.100`을 빼는 패턴이
  자주 보인다. 문턱 경계에서 판단이 갈리므로 목적은 달성했지만, 프롬프트가
  **재추론**을 유도하는 정도를 openai에서 재확인.

## 4. 며칠 돌려보고 판단할 것 (코드 작업 아님)

### 4-1. 인사이더 투표 빈도 — 문턱 재조정

`insider_scoring`은 **대부분의 날 조용하다**. 픽의 12~14%만 Form 4를 내고,
그중 재량 거래(P·S ∧ ¬10b5-1)는 다시 일부다 — 실측 거래 79건 중 2건.

잡 원장 detail에 `N insider votes, M abstained`가 남으므로, 며칠 쌓인 뒤
**투표가 실제로 얼마나 자주 서는지** 보고 `mvp2.insider`의 문턱(buy 0.80 /
sell 0.30)을 조정할 것. **지금 표본은 하루치뿐이다.**

### 4-2. 갭 가드 문턱 — `premarket_gap_max` 0.03

⏳ 잠정값. 실제 갭 분포를 1~2주 관측한 뒤 조정. 발동 사유는
`tb_order_plan`에 기록되므로 사후에 셀 수 있다.

### 4-3. 무인 완주

`jobs.enabled=true`는 2026-07-20에 켜졌지만 **사람 없이 하루를 완주한
