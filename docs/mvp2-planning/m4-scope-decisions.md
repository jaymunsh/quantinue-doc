## 결정

### D1. role_05 CIK 실배선을 M4 최우선으로 올린다 (신규 4-0)

**결정**: 신규 태스크 `4-0`으로 분리해 **가장 먼저** 처리.

**근거**: F1은 단순 버그가 아니라 **이미 완료 처리한 4-2 게이트를 무력화**한다. `hard_negative_max`(공시 0.15 이하 매수 차단)가 남의 회사 공시로 발동/미발동하고, 07의 4표 중 1표가 거짓이다. 신규 방어선을 얹기 전에 기존 방어선의 입력을 먼저 진실하게 만들어야 한다.

**뒤집을 조건**: 없음. 무조건 선행.

### D2. 4-5 대표기사 — "importance 최상위"를 "신뢰가중 최상위 + peak=max(importance)"로 개정

**결정 (A안)**:
- 선정 랭킹 = `relevance ≥ MINIMUM_RELEVANCE_SCORE` 통과 필터 → `confidence × grade_weight` 최상위
- `peak_importance` = **LLM importance의 사이클 내 러닝 맥스**. 저장은 `on_conflict_do_update` + `greatest()`

**근거**: F2 때문에 playbook 원문("importance × 신뢰무게 최상위")은 그대로는 구현 불가.
검토한 B안(상위 K건을 전부 LLM에 태워 importance 최댓값 선택)은 뉴스 토큰이 K배 → M8 일일 예산 가드($3/일)와 정면충돌. 소규모 프로젝트에서 감당할 트레이드가 아니다.

**뒤집을 조건**: 뉴스 LLM 비용이 실측상 예산 대비 충분히 낮고(M8-6 비용표 확정 후), 대표기사 오선정이 실제 오판으로 이어진 사례가 관측되면 B안 재검토.

### D3. 4-6 Form 4 — 범위를 "오인 차단 커널"로 축소, 본체는 백로그

**결정**:
- **M4에 남기는 것(커널)**: `form_type == '4'` → LLM 우회 + 결정적 저importance 부여
- **백로그로 내보내는 것(본체)**: ownership XML fetch·파서, 코드 P 판별, 임원·$문턱, 클러스터(2인+) 가산/매도 리스크

**근거 3가지**:
1. **시그널 지평 불일치** — 인사이더 매수의 초과수익은 수 주~수 개월에 걸쳐 나오는데 이 시스템은 **T+5 채점**이다. 지평이 안 맞는 시그널을 넣으면 M7 학습 루프가 계속 "틀렸다"고 채점하고, 그 lesson이 07 프롬프트를 오염시킨다.
2. **비용이 설명과 다르다** — playbook은 "LLM 우회 분기"라 적혀 한 시간짜리로 읽히지만, F3 때문에 신규 fetch + 파서 + 경계 모델이 필요하다. 게다가 클러스터 판정은 `role_05`를 "최근 1건" → "최근 N건 윈도우"로 바꾸는 구조 변경을 동반한다(M5급).
3. **성격이 섞여 있다** — 클러스터 **매도**는 리스크 시그널(방어)이 맞지만, 매수 스코어링은 알파다. 작업량 대부분이 알파 쪽에 붙는다.

**커널을 남기는 이유**: 지금은 `form_type`이 `"8-K"`로 하드코딩(`service.py:138`)돼 form 구분 자체가 없다. **Form 4를 8-K로 오인해 중대 이벤트로 채점하고 매수 근거로 쓰는 것**을 막는 건 명백히 방어선이고, 몇 줄이면 된다.

> ※ 초안에서 "4-6은 방어선이 아니므로 통째로 뺀다 / M4를 7건으로 닫는다"고 했으나 **철회**. 개념 자체는 멀쩡하고, 정형 문서를 LLM에 안 태운다는 설계 판단도 옳다. 문제는 배치와 견적이었다. **M4는 8건 그대로 닫는다.**

**뒤집을 조건**: 보유 기간 정책이 T+5보다 길어지면(M5 시간청산 `exits.time_exit_bdays`=10영업일 기준으로 재평가) 지평 불일치 논거가 약해진다. 백로그 착수 전 **정합성 재검토를 ⏳로 명시**.

### D4. signal_consensus CHECK를 0~4로 넓힌다

**결정**: 마이그레이션으로 `CHECK (signal_consensus BETWEEN 0 AND 4)`. role_07이 계산해 하향 전달(DB 계층 계산 불가 — `StrategistSignalWrite`에 technical/model 점수가 없다).

**근거**: 투표원이 실제로 4개(F4). 마이그레이션 한 줄이 "3개로 줄여 정의하기" 논쟁보다 싸다. 게이트가 아니라 **저장 전용 관측 지표**이므로 행동 변화 없음.

**뒤집을 조건**: 없음.

### D5. 개장 후로 미룰 코드 작업은 없다

**결정**: 8건 전부 오늘 처리.

**근거**: 초안에서 4-7a·4-5를 "개장 후"로 뒀으나 **기술적 의존성이 없다**(우선순위였지 선후관계가 아니었다). A안 확정으로 4-5도 실측이 불필요해졌다.

**진짜로 개장 후에만 되는 것 = 코드가 아니라 관측 2건**:
- `premarket_gap_max` 3%의 적정성 — 실제 갭 분포 관찰 필요. 지금 뭘 해도 앞당길 수 없다.
- "block 매체 LLM 호출 0" 실측 — M4 완료 기준에 미검증으로 남아 있던 항목. 드라이런 1회로 확인.

- **4-5** — `importance·peak_importance·source_trust·grade_score` 전부 채워짐(이전 행은 전부 NULL). `source_trust=0.5`(gray) → 4-4의 0.55 문턱을 못 넘어 투표권 상실 — **두 층의 맞물림이 실데이터에서 동작**

**확인 못 한 것 (정직하게)**:
- **갭 가드·late_entry·halted 미발동** — 크리틱이 먼저 차단(`conviction 0.218`)해 게이트 체인이 `critic_rejected`에서 끊겼고, mock 브로커는 `is_tradable`을 구현하지 않는다. **유닛으로만 검증된 상태**로 남는다.
- **차단매체 LLM 호출 0** — 필터는 실행됐으나(`차단매체 0건 제외`) 이번 표본(AAPL 구글뉴스 100건)에 차단 도메인이 없었다. **메커니즘이 돌았을 뿐 차단이 일어난 것은 아니다** — ⏳ 유지.

### 배운 것

**"부재를 0점으로 쓰지 마라"가 M4 전체를 관통하는 규칙이 됐다.** 공시 부재(4-0)·소유권 신고서(4-6)·저신뢰 뉴스(4-4, 기완료) 모두 같은 함정이었다 — 0점은 중립이 아니라 **강한 악재**로 읽히고, `hard_negative_max`에 걸려 매수를 차단한다. 앞으로 새 신호를 붙일 때 **"모르는 상태"를 표현할 자리를 먼저 만들 것**.

**유닛 그린이 통합 그린을 뜻하지 않고, 통합 그린이 실행 그린을 뜻하지 않는다.** F11은 유닛 649건이 전부 통과한 뒤 통합에서만 터졌고, F13·F14는 유닛·통합을 모두 통과한 뒤 **실행에서만** 터졌다. mock transport는 상대 서버의 정책(UA 요구·거부)을 흉내내지 않는다. **새 외부 엔드포인트를 붙이면 반드시 실호출로 한 번 찔러볼 것.**

**두 상류의 요구가 반대일 수 있다.** F14가 특히 그렇다 — SEC는 연락처를 요구하고 NASDAQ은 연락처를 거부한다. 전역 설정으로 외부 규약을 맞추려는 시도는 소스가 둘 이상이면 언제든 이렇게 깨진다. **소스별 정책은 소스별로.**
